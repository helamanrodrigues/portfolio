Tutorial GIT IT TOGETHER

Here are some instructions to not forget how to make the conection between the editor (Virtual Studio Code) and the Repository on GitHub:

  1. The first thing you need to do it enter on your GitHub account and create a new Repository
  2. After creating your Repository, go to your editor that in my case is Virtual Studio Code and open the folder that you will use!
  3. Check in your folder if you have the basic of the code like some of HTML, CSS and JS. If you don't have nothing, I suggest that you create at least one.
  4. Open your terminal, and search for this folder that you opened on VS. You can write ls -al to check with your folder is inside to the point that you are.
  5. Write cd "name of the folder". After the folder is opened in the terminal, write ls -al again to verify if is the correct folder.
  6. If the folder that is opened in the terminal is the correct one, write "git init" to start the git on the folder.
  7. Write ls -al again to verify if the .git was added inside your folder.
  8. In your github account, copy the url link of your Repository.
  9. In your terminal write git remote add origin "the url link of your Repository" and press enter
  10. Write git remove -v to check if has the fetch and push with your url link
  11. Write git status to check what you need to do, that will be in this first case to use the command git add. Note: Here we are will using the signal "." after the add, that means that we will add all the things that have in the folder.
  12. Write again git status to check that now we will need to commit and use the command git command.
  13. Write the command git commit -m and write a message in quotes. Note that the signal -m means message, always when you will commit, you need to write what you are doing, this helps to track your way.
  14. Write again git status and you will check that now it is okay. So here we can continue for the next step, that is the git push.
  15. Write git push origin master and press enter, after this you need to write your github username and password (token).
  16. Update your Repository on GitHub and you will check that the folder is there.
