Getting Started with JSON Web Tokens Complete:

What is JSON Web Token?

  • Is a compact standard that is used to share and transmits information in a safe way.
  • JSON Web Tokens are most use for two things: Authentication (an example could be a log in to an account) and Information exchange (This helps to securely exchange information to one another. Some examples could be banks, schools, government jobs).

The main three parts of the JWT are:

-Header (Two parts: the type and the hashing algorithm)

Example: { "alg": "HS256", "typ": "JWT" }

Payload (There are three types of claims: Reserved, Public and Private)

Example: { "sub": "1234567890", "name": "John Doe", "admin": true }

Signatures.

How JSON Web Tokens Work?

Note: A good practice is not to store data for a long time in the browser storage because of lack of security.

How JSON Web Tokens Work on my own words:

The browser makes the request, the server sends a Web Token, The Computer Sends a JWT on the authorization header. Finally, the server checks the signatures and sends the information to the client. In my opinion we just have to be careful with specifying the parts of the of the JSON Web Tokens.
