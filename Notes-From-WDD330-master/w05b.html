  • Variables are spaces in the global memory that store a specific value or are undefined.
  • Functions: are meant perform a specific action, they use variables that are declared inside the function or in the global memory. They follow an order of instructions "remembering the values of the variables" to create an output.
  • Execution: when the functions are called. But it is actually the process of jumping back an forth to the global memory to use the variables and perform an action that will bring an specific value
  • A local memory (Variable environment) where anything defined in the function is moved: At the end, we have to keep track or our function calls to keep track of the values and to return a correct result.
  • Call Stacks: Keeps the order starting from the global environment on how each function was called. This process helps us to have an order and a way to keep track of the value of the variables.
